Los estudiantes podrán: evaluar, depurar y mejorar la lógica de un programa, usando Delightex para probar cambios, y explicar por qué el control humano es clave ante errores de IA.

                    
                
                
                    

🤖 Prompt de Referencia para XOLO (Solo para Maestro)

                    
                        
                            ⚠️ NOTA: XOLO ya conoce el tema de esta misión. Este prompt es SOLO de referencia en caso de emergencia técnica:                                                      
                        
Explícame qué significa que una IA “alucine” en código: puede sonar correcto pero tener errores sutiles o decisiones ineficientes. Guíame para revisar críticamente: lógica, variables, condiciones, repetición innecesaria, activaciones erróneas. Acompáñame a crear una lista de verificación de optimización y pruebas finales, enfatizando que el juicio humano es indispensable.
                    
                
                
                    

⚡ Puntos Esenciales de la Misión

                    
                        
                            
🎓
                            

Concepto Clave

                            

Falibilidad de la IA: Es la posibilidad de que la inteligencia artificial cometa errores que parecen correctos pero no lo son, como un amigo que te da una dirección con mucha seguridad pero te lleva al lugar equivocado.

                        
                        
                            
🔧
                            

Habilidad Técnica

                            

Usar Delightex para identificar y corregir errores sutiles en la lógica de programación.

                        
                        
                            
🧪
                            

Experimentación

                            

Probar con diferentes escenarios de error y condiciones lógicas para entender limitaciones.

📊 Diapositiva 3: Brain Spark

                    
                        

1. 💬 Pide a XOLO que te ayude a crear:

                        
                            
"Ayúdame a revisar mi proyecto final buscando posibles fallos o ineficiencias en el código o lógica generada con IA. Guíame para identificar 3 riesgos típicos (errores sutiles, lógica incompleta, activaciones inesperadas) y proponer ajustes. Después ayúdame a definir una ronda de pruebas finales para validar estabilidad."
                        
                        

2. 🔬 Experimenta y descubre con tu programa:

                        
                                
  • ¿El código funciona exactamente como esperabas en todas las situaciones?
    •                             
  • ¿Qué pasa si intentas activar una función en un orden inesperado en Delightex?
    •                             
  • ¿El programa responde igual de rápido después de eliminar las repeticiones innecesarias?
    •                             
  • ¿Cómo demuestra este error que la IA no es perfecta y puede fallar (falibilidad)?
    •                             
  • ¿Por qué es vital que tú, como humano, tomes la decisión final sobre qué código dejar (control humano)?
    •                         
                    
                
                
                                         
                        

¡Es hora de crear y perfeccionar! Vamos a pasar de la teoría a la práctica. El flujo de hoy es: idear la revisión con XOLO, definir el VIBE de su proyecto, oobtener el GCS (Gnius Code Spark), aplicarlo en Delightex y experimentar. Por ejemplo, podrían estar revisando un mundo virtual donde un objeto no aparece cuando debería. Usen el prompt de la pantalla para que XOLO los ayude a encontrar esos 3 riesgos típicos. No olviden las preguntas de experimentación para validar su estabilidad. Al terminar, entreguen sus equipos con cuidado. Si tienen errores o dudas, pregúntenle a XOLO, él los ayudará a depurar.
